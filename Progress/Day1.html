From my understanding I'll only need one app for this project. For a minute I was wondering if I would need to create an app for creating a user, posting, commenting, etc., but from the documentation and tutorials I'm looking at it appears that I can do that all in one. We'll see!

Models

This system will need three basic models:

  • *Post
  • *Comment
  • User
  • PM

*I need to figure out where in the framework I'm going to put the URL recognition and URL to image import.

Thankfully I don't need to do all of the bullet points on the list, so I'll triage and plan out my app creation in bite size chunks. I guess I could look at this project as an iteration and then divide the project into sprints.

Views

According to the blog by Chris Bartos I mentioned in yesterdays post, each View is a verb. From my experience, this seems accurate, so now I will break out the app specs into a variety of views:

  • User interaction with account
    • Sign Up
    • Login
    • Logout
    • Delete Account
  • Sort and search
    • View all posts
    • View posts by user*
    • Search posts**
  • Logged on user capabilities
    • Create post
    • Edit post
    • Create comment
    • Edit comment

*I will need to make decisions about how the view will display post and/or comments by user. Just do posts? If I do posts and comments should I include parent post? TBD later.
